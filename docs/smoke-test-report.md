## 二、测试中发现并已修复的问题

### 1. 供应商编号首次创建报错（已修）
- **现象**：当日第一个供应商报「编号已用尽」
- **原因**：`INSERT` 时未用 `LAST_INSERT_ID(1)`，导致 `SELECT LAST_INSERT_ID()` 返回 0
- **修复**：`CustomerIdSequenceMapper` / `SupplierIdSequenceMapper`

### 2. 客户编号重复 500 错误（已修）
- **现象**：连续新建客户时偶发 `Duplicate entry '1-20260625004'`
- **原因**：序列表 `last_seq` 与已有 `customer_no` 不同步
- **修复**：编号生成前先从数据库同步当日最大序号

### 3. 8080 旧进程缺少新接口（需操作）
- **现象**：供应商报表、库存报表返回 500
- **原因**：8080 进程未重启，未加载新 Controller
- **处理**：重启 `mvn spring-boot:run` 后再测小程序
